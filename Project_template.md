# Архитектура монолитного приложения:
  - Язык программирования: Java
  - База данных: PostgreSQL
  - Архитектура: Монолитная, все компоненты системы (обработка запросов, бизнес-логика, работа с данными) находятся в рамках одного приложения.
  - Взаимодействие: Синхронное, запросы обрабатываются последовательно.

### **4. Проблемы монолитного решения**

- Пользователи не могут самостоятельно подключить свой датчик к системе.
Это огромгные затраты на ресурсы - установщики, и затраты времени. Так абонентскую базу не увеличишь.
- Ограничена масштабируемость, так как монолитное приложение сложно масштабировать по частям.
- Только синхронное взаимодействие
- Отсутствие раективного взаимодействия
- Одностронее взаимодействие 
  - Управление (сервер -> датчик)
  - Снятие показаний (сервер -> датчик)

Если вы считаете, что текущее решение не вызывает проблем, аргументируйте свою позицию.
